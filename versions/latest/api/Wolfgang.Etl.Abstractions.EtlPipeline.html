Entry point for building a generic, format-agnostic ETL pipeline. Obtain a fresh builder with Create(), start from a source — either a built-in From factory (see EtlPipelineSourceExtensions) or a format-specific factory hung off the EtlPipeline instance by a format package — append transformer stages on IEtlPipeline<T>, terminate with a sink, then call RunAsync(IProgress<EtlPipelineProgress>?, CancellationToken).

    .RunAsync(progress, token);

Remarks

The name EtlPipeline (rather than Pipeline) avoids clashing with the fluent Pipeline extract/transform/load builder in this same namespace and with System.IO.Pipelines.

Create() returns a fresh EtlPipeline seed: it carries no data itself and exists to give source factories a strongly-typed receiver. The built-in From factories and any format-package factories are extension methods on this type — for example a CSV package adds public static ICsvExtractorBuilder<T> CsvExtractor<T>(this EtlPipeline pipeline, string path), enabling EtlPipeline.Create().CsvExtractor<Order>("orders.csv").

The core exposes only the plumbing — a source, Through<TOut>(ITransformAsync<T, TOut>) for appending transformer stages, and a sink. The LINQ-flavored operators (Where, Select, Distinct, …) are extension methods shipped by Wolfgang.Etl.Transformers, which already owns those transformers.
