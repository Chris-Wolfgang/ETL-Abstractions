A generic, format-agnostic ETL pipeline carrying a stream of items of type T. Append transformer stages with Through<TOut>(ITransformAsync<T, TOut>), then terminate with To<TProgress>(LoaderBase<T, TProgress>) (or a format-specific sink terminator) to obtain a runnable IEtlPipelineSink.

public interface IEtlPipeline<T> where T : notnull

Type Parameters

T

The type of item currently flowing through the pipeline.

Remarks

This is the minimal pipeline core. It knows how to pull from a source, pipe through transformers, and push to a sink — nothing more. The LINQ-flavored operators (Where, Select, Distinct, Take, Buffer, …) are extension methods that layer on top of Through<TOut>(ITransformAsync<T, TOut>) and are shipped by Wolfgang.Etl.Transformers, which already owns those transformers — the core does not re-implement them.

The pipeline is lazy: appending a stage builds up a description of the work and nothing executes until RunAsync(IProgress<EtlPipelineProgress>?, CancellationToken) is called (or the stream is enumerated via AsAsyncEnumerable(CancellationToken)).

Through<TOut>(Func<IAsyncEnumerable<T>, IAsyncEnumerable<TOut>>)

Appends a stream-to-stream transformer stage supplied as a delegate — the same contract as TransformAsync(IAsyncEnumerable<TSource>), but inline, without declaring a class. Note this transforms the whole stream; a per-element projection (Select) is an operator provided by Wolfgang.Etl.Transformers, not the core.

To<TProgress>(LoaderBase<T, TProgress>)

Terminates the pipeline with a loader. Format-specific sink terminators (for example CsvLoader or SqlBulkCopyLoader) are extension methods shipped by their own packages; this generic overload is the catch-all for callers who already hold a LoaderBase<TDestination, TProgress>.
